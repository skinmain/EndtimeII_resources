# 自定义结构
### 须知  
- 投稿后，文件会经作者和管理员（目前还没有）审查、记录。  
- 当通过的投稿足够多时，会统一整合进后续发布的整合包。  
- 若投稿存在微小问题（如命名不规范、盐值重复、生成不合理等），作者会提前通知你，并在你的同意下进行微调。
- 投稿前，建议先学习[数据包自定义结构教程](https://zh.minecraft.wiki/w/Tutorial:%E8%87%AA%E5%AE%9A%E4%B9%89%E7%BB%93%E6%9E%84%E7%94%9F%E6%88%90)。


### 投稿要求
- 符合 **[内容发布准则](rule.md)** 的所有要求
- 建筑 / 结构需符合本整合包的世界观与剧情设定。
- 除非 其他建筑生成的同类战利品太少（目前应该没有）/探索代价高/生成密度过于稀疏（平均每60个区块及以上生成一次），否则 ***不建议*** 让建筑生成太多战利品，防止游戏体验崩坏。
- 整体风格应贴近 **60–90 年代东欧地区建筑风格**（可根据实际建造内容适度微调，不破坏整体观感即可）。<br><br>

### 打包方式： 
- 将你测试好的数据包核心部分（即 **[数据包名称]/data/** 目录下以你的**命名空间**命名的**整个文件夹**，详见 [Minecraft Wiki：数据包内容](https://zh.minecraft.wiki/w/%E6%95%B0%E6%8D%AE%E5%8C%85#%E6%95%B0%E6%8D%AE%E5%8C%85%E5%86%85%E5%AE%B9)）压缩打包  
- 文件夹命名应能**标识制作者身份**且能**明显区别于他人**（不要使用无意义名称）。
- 压缩格式支持 **.zip、.rar、.7z** 等主流格式。  
- 你也可以将内容直接制作成*完整数据包*（**包名和主命名空间**请使用能**标识制作者身份**的命名），然后上传。  
>***⚠禁止上传恶意数据包！***

### 投稿描述
描述中请务必包含以下信息：

- 是否适配本整合包已发布的最新版本
- 结构的具体内容
- 配置文件说明（如*生成间隔、自定义战利品表*等）
- 生成效果及部分内饰截图
- *（可选）其他联系方式，方便沟通*

描述越清晰、简洁、详细，审核优先级越高。
